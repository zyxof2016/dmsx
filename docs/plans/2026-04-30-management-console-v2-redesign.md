# DMSX 管理台 v2 重设计方案

## 目标

当前管理台的主要问题不是缺少页面，而是页面职责、导航分组、权限维护、设备生命周期和平台治理混在一起。v2 设计允许推倒现有前端组织方式，但保留已经可用的后端能力作为业务资产：认证、多租户、设备、命令、策略、制品、合规、审计、远程桌面、平台租户目录、平台配额、平台健康。

v2 的成功标准：

- 新用户登录后能明确知道自己在“平台管理”还是“租户管理”。
- 平台管理员能在一个权限中心维护平台用户、角色、菜单和策略，不需要猜入口。
- 租户用户按设备生命周期工作：接入设备、分组筛选、查看状态、下发作业、升级应用、看审计和合规。
- 所有主页面采用一致的信息密度：摘要区 + 筛选条 + 单主列表 + 抽屉详情 / 编辑。
- 所有页面都能解释空态、加载态、错误态和无权限态。

## 设计决策

### 决策 1：保留后端能力，重建前端产品模型

不删除现有 API，不先重写后端；前端先按 v2 产品模型重新组合页面。原因是当前后端已经支持核心链路，真正混乱在“能力如何呈现”。后续缺口再补 API，例如平台权限 CRUD、Fleet 管理、Job / OTA。

### 决策 2：三层导航，而不是路由平铺

导航层级固定为：

1. 工作范围：平台管理 / 租户管理。
2. 业务模块：例如权限中心、运营治理、设备运营、策略安全。
3. 对象页面：角色、用户、菜单、设备、作业、制品等。

顶部只保留全局动作：搜索、刷新、语言、主题、用户菜单。左侧只承载业务模块，不再放杂项工具入口。

### 决策 3：对象页统一采用“列表 + 抽屉”

角色、用户、菜单、租户、设备、策略、制品、审计、作业都采用同一种结构：

- 顶部摘要：最多 3-4 个关键指标。
- 筛选条：搜索、状态、类型、时间范围、标签。
- 主列表：只展示做决策需要的信息。
- 抽屉：详情、编辑、危险操作、关联对象。

这样用户不用在不同页面重新学习交互。

## v2 信息架构

### 平台管理

| 模块 | 页面 | 必备能力 |
|------|------|----------|
| 平台工作台 | 总览 | 租户数、设备数、活跃命令、配额风险、组件健康、待处理事项 |
| 权限中心 | 用户 / 角色 / 菜单 / 策略 / 审计 | 平台用户目录、角色绑定、菜单权限映射、策略版本、权限变更审计 |
| 租户运营 | 租户 / 配额 / Profile | 租户创建、停启用、配额维护、租户模板、默认限制 |
| 平台运维 | 健康 / 全局审计 / 系统设置 | LiveKit、Redis、NATS、Postgres、配置、跨租户审计 |

### 租户管理

| 模块 | 页面 | 必备能力 |
|------|------|----------|
| 工作台 | 态势 / 告警 / AI | 租户内关键指标、风险、待执行作业、AI 辅助分析 |
| 设备运营 | 设备 / 分组 / 接入批次 / 远程桌面 | 预注册、零接触安装、分组、标签、设备详情、影子、远控 |
| 作业中心 | 作业 / 命令 / OTA | 批量脚本、升级、重启、证书轮换、灰度、失败阈值、重试 |
| 策略安全 | 策略 / 合规 / 租户用户角色 / 审计 | 策略版本、合规发现、租户 RBAC、审计查询 |
| 交付配置 | 制品 / 网络 / Device Profile | Agent 制品、安装元数据、网络站点、设备类型默认配置 |

## 主流程

### 登录与范围选择

1. 用户输入账号密码。
2. 后端返回可进入范围：平台、租户列表、每个租户有效角色。
3. 前端展示工作范围选择页。
4. 用户进入平台或某个租户后，左侧菜单只显示该范围可用模块。

### 设备接入

1. 进入租户管理 -> 设备运营 -> 接入批次。
2. 选择 Windows / Linux / Android / 手工注册。
3. 平台生成批次、设备预注册记录、安装包或安装命令。
4. Agent 首次启动自动注册。
5. 用户在批次详情中查看成功、失败、待上线设备。

### 批量运维

1. 在设备列表通过分组、标签、版本、在线状态筛选目标。
2. 保存为静态目标集合或动态查询。
3. 创建 Job：脚本、升级、重启、证书轮换或诊断。
4. 配置 rollout、失败阈值、超时、重试。
5. 在 Job 详情中看执行聚合与单设备结果。

## UI 规范

- 布局：左侧 240px 模块导航，顶部 48-56px 工具栏，内容区 16px 外边距。
- 页面标题：只显示对象名和范围信息，不写大段说明文。
- 主色：保留蓝色作为操作色，状态色遵循 Ant Design 语义色。
- 卡片：只用于摘要指标、重复对象或抽屉内分区，不把页面区块层层套卡。
- 表格：默认密度偏紧凑，固定操作列，长文本进入抽屉。
- 空态：必须给下一步动作，例如“创建设备接入批次”。
- 错误态：展示接口错误摘要和重试按钮。
- 无权限态：说明缺少的平台/租户角色，并提供返回当前范围首页。

## 实施计划

1. 抽离导航模型：把菜单、权限、分组、搜索入口从 `App.tsx` 移到独立配置。
2. 重建应用壳：左侧模块导航、顶部全局工具、访问标签、范围选择保持一致。
3. 重做平台权限中心：先把当前 UI 收敛为对象页，再补后端 CRUD。
4. 重做设备运营：设备列表、接入批次、分组和设备详情统一为 Fleet 工作流。
5. 新增作业中心：先用现有 command / install_update 承载最小 Job UI，再补后端 Job 模型。
6. 收尾一致性：统一抽屉、筛选条、摘要卡、空态、错误态、权限态组件。
